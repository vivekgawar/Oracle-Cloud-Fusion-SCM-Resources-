# Understanding Your Oracle Fusion Cloud SCM Training: A Module-by-Module Guide for a New Accenture AASE

## TL;DR
- This program trains you to be an Oracle Fusion Cloud SCM **technical consultant** — someone who configures, reports on, extends, integrates, and now AI-enables Oracle's cloud supply-chain software for client companies. It deliberately moves through layers: cloud infrastructure foundations → core SCM business knowledge → reporting → extensibility/customization → integration → an AI agent layer.
- The 13 modules form a coherent arc: you first learn *where the software runs* (Autonomous Database, OCI), then *what the business software does* (SCM Foundation, Process Essentials, Setup/Security), then *how to get data out and present it* (OTBI/BIP/OAC), then *how to change the software safely* (Redwood, REST, Groovy), then *how to connect it to other systems* (XML, OIC), and finally *how to layer AI agents on top* (GenWizard, AI Agent Studio, AI Refinery).
- As an Advanced Associate Software Engineer (AASE), you'll be the early-career engineer who builds the technical artifacts on Fusion SCM projects — reports, integrations, page extensions, data loads, and increasingly AI agents — under guidance, which is exactly why the curriculum blends functional, technical, integration, and AI skills.

## Key Findings

**What the AASE role is.** Advanced Associate Software Engineer is Accenture's early-career engineering title (a step up from Associate Software Engineer). On an Oracle Fusion SCM project, this person is a "techno-functional" consultant: they need enough functional knowledge of supply-chain processes to understand client requirements, plus the technical skills (reporting, integration, extensibility, scripting) to actually build solutions. Accenture is explicitly steering these roles toward AI-driven delivery, which is why the program ends with three AI-centric modules.

**The overall arc.** The program is sequenced from infrastructure upward. Modules 1–2 establish the cloud platform. Modules 3–5 build functional SCM fluency and the administrative/security backbone. Module 6 covers reporting. Module 7 covers safe customization. Modules 9–11 cover integration (with XML as the data-format prerequisite). Modules 8, 12, and 13 add the modern AI layer. A 5-day capstone ties it together.

**Acronym glossary.** SCM = Supply Chain Management; OCI = Oracle Cloud Infrastructure; ADB = Autonomous Database; FSM = Functional Setup Manager; OTBI = Oracle Transactional Business Intelligence; BIP/BI Publisher = Business Intelligence Publisher (now Oracle Analytics Publisher); OAC = Oracle Analytics Cloud; BICC = BI Cloud Connector; FBDI = File-Based Data Import; HDL = HCM Data Loader; OIC = Oracle Integration Cloud; OMBP = Oracle Modern Best Practices; REST API = Representational State Transfer Application Programming Interface; XSLT = Extensible Stylesheet Language Transformations; XQuery = XML Query language; XPath = XML Path Language; DTD = Document Type Definition; TDLC = Technology Delivery Life Cycle; RBAC = Role-Based Access Control; LLM = Large Language Model; RAG = Retrieval-Augmented Generation; MCP = Model Context Protocol; A2A = Agent2Agent protocol; VBCS/VB Studio = Visual Builder (Cloud Service) Studio.

## Details

### Cluster 1 — Cloud Infrastructure Foundations

**Module 1: Oracle Autonomous Database Cloud (18 hrs).**
*(a) What it is:* A database is the system that stores all the structured business data (orders, inventory, suppliers). Oracle Autonomous Database (ADB) is Oracle's flagship cloud database that is "self-driving, self-securing, and self-repairing" — it uses machine learning to automate the tasks a human database administrator used to do (provisioning, patching, tuning, indexing, backups, scaling) while the system keeps running.
*(b) Topics covered:* Provisioning an Autonomous Database; the two workload flavors — Autonomous Data Warehouse (ADW, for analytics) and Autonomous Transaction Processing (ATP, for transactional apps); serverless vs. dedicated deployment; elastic scaling of compute and storage; data security (Transparent Data Encryption at rest, TLS in transit, automatic patching); and performance tuning (automatic indexing, real-time statistics). It runs on Oracle Exadata infrastructure.
*(c) Why it matters:* Everything in Oracle Cloud ultimately sits on a database. With your AI/ML background, this is your most familiar entry point — note that the "self-driving" automation is itself powered by ML. It's foundational to the analytics and integration work later in the program.

**Module 2: OCI Foundation (27 hrs).**
*(a) What it is:* Oracle Cloud Infrastructure (OCI) is Oracle's public cloud — the data centers, servers, networking, and storage that compete with AWS and Azure. This module maps to the Oracle Cloud Infrastructure 2025 Foundations Associate certification, exam 1Z0-1085-25 (per Oracle University: 90 minutes, 55 multiple-choice questions, 68% passing score).
*(b) Topics covered:* Core architecture concepts — **Regions** (geographic locations), **Availability Domains** (isolated data centers within a region), **Fault Domains** (hardware groupings within an AD for failure isolation); **Tenancy** (your cloud account) and **Compartments** (logical containers to organize and secure resources); **IAM** (Identity and Access Management — users, groups, policies, dynamic groups; authentication vs. authorization); networking (Virtual Cloud Networks, subnets, gateways); compute; and the four storage types (block, object, file, archive). The "distributed cloud model" refers to OCI's options like Dedicated Region and Cloud@Customer.
*(c) Why it matters:* Fusion SaaS applications run on OCI, and the integration/AI tools you'll use (OIC, AI services) are OCI services. Understanding compartments, IAM policies, and regions is essential for any technical work and for client security conversations.

### Cluster 2 — Core SCM Functional Knowledge

**Module 3: Supply Chain Management Foundation (27 hrs).**
*(a) What it is:* Your first exposure to the actual supply-chain business software. "Supply chain" = the end-to-end flow of getting products from suppliers, through your warehouses, to customers. This module covers the common plumbing all SCM modules share.
*(b) Topics covered:* Common setup tasks (enterprise structures, inventory organizations); basic inventory tasks; **inventory transactions** (miscellaneous receipts/issues, subinventory transfers, interorganization transfers, min-max replenishment); and supply chain managerial accounting / **cost management** (cost organizations, cost books, standard vs. average costing, landed cost). Key jargon: an *item* is a product master record; an *inventory organization* is a warehouse/plant; a *subinventory* is a storage area within it; a *locator* is a specific bin.
*(c) Why it matters:* This is the vocabulary layer. You can't write a meaningful inventory report or integration without knowing what a subinventory transfer or a cost book is. It underpins every other functional module.

**Module 4: SCM Process Essentials (36 hrs).**
*(a) What it is:* A broader tour of the four functional pillars of Fusion SCM, framed around **Oracle Modern Best Practices (OMBP)** — Oracle's library of recommended, increasingly AI/ML-powered business processes. Per Oracle's official learning-path description, it "introduces Oracle Modern Best Practices (OMBP) for Supply Chain Management (SCM), showing how AI- and ML-powered processes in Oracle Fusion SCM optimize essential modules such as Planning, Procurement, Order Management, and Logistics."
*(b) Topics covered:* The four pillars — **Planning** (demand management, supply planning, replenishment, backlog management), **Procurement** (the source-to-pay / procure-to-pay process, sourcing, supplier qualification, procurement contracts), **Order Management** (the order-to-cash process), and **Logistics** (warehouse management, transportation management, global trade management). It uses SCM product demos, **key SCM metrics**, and design insights, and covers the Oracle Cloud Success Navigator. A central concept is **OMBP (Order Management Business Process / Oracle Modern Best Practices)** and the **order-to-cash orchestration** flow: an order is captured, then an *orchestration process* runs steps (schedule → reserve → ship → invoice), calling Global Order Promising for availability and Inventory/Receivables for fulfillment and billing. It prepares you for the Process Essentials certification (exam 1Z0-1163).
*(c) Why it matters:* This is the conceptual map of the whole SCM suite. The order-to-cash orchestration concept in particular recurs in integration and AI modules.

**Module 5: Set Up, Roles & Security in Oracle SCM Foundations (18 hrs).**
*(a) What it is:* How Fusion applications are configured and secured — the administrator's toolkit.
*(b) Topics covered:* SaaS administrative applications; the **Functional Setup Manager (FSM)** — the single tool where all configuration happens via Offerings → Functional Areas → Features → Tasks, organized into Implementation Projects, with setup data moved between environments via Configuration Packages; and core **Security** based on **Role-Based Access Control (RBAC)**. Fusion's role model has four layers: **Abstract roles** (general, e.g., Employee), **Job roles** (job titles, e.g., Warehouse Manager), **Duty roles** (granular task bundles where security policies attach), and **Data roles** (which business units/data a user can see). The module covers customizing/copying security roles.
*(c) Why it matters:* FSM is where consultants spend much of their time; security is a top client concern. Reports and integrations also inherit this security model, so you must understand it before building them.

### Cluster 3 — Reporting and Analytics

**Module 6: Reporting and Analytics in Oracle SCM (36 hrs).**
*(a) What it is:* The tools to extract data from Fusion and present it as reports, dashboards, and analytics.
*(b) Topics covered:* **OTBI (Oracle Transactional Business Intelligence)** — real-time, drag-and-drop, self-service analytics built on predefined "subject areas" that query live transactional data and respect Fusion security; best for ad-hoc operational analysis within a single subject area. **BI Publisher / Oracle Analytics Publisher (BIP)** — "pixel-perfect," formatted, template-based reporting (invoices, POs, checks) that can run custom SQL, schedule, and "burst" outputs to many recipients; best for high-volume, formatted, or scheduled output. **OAC (Oracle Analytics Cloud)** — a separate enterprise analytics platform for data modeling, visualization, and advanced/predictive analytics across sources. Plus **data extraction tools**: **BICC (BI Cloud Connector)** for bulk *outbound* extraction of Fusion data (via Public View Objects) into CSV for data warehouses; and inbound data tools **FBDI (File-Based Data Import)** — bulk import into Fusion ERP/SCM via Excel/CSV templates — and **HDL (HCM Data Loader)**, the HCM equivalent using pipe-delimited files.
*(c) Why it matters:* Reporting is often an AASE's first real deliverable. Knowing when to use OTBI vs. BIP vs. OAC vs. BICC is a core consultant judgment. These tools also overlap with integration (BIP and BICC are used to extract data for downstream systems).

### Cluster 4 — Extensibility and Customization

**Module 7: Redwood and Extensibility (18 hrs).**
*(a) What it is:* How to change the look and behavior of Fusion without breaking Oracle's quarterly upgrades. **Redwood** is Oracle's modern design system and application-building platform now standard across Fusion; it's not just a visual refresh but includes search, embedded AI, business rules, and guided journeys.
*(b) Topics covered:* The Redwood UX/design system; extensibility tools — **Visual Builder Studio (VB Studio) Express Mode**, which is the supported page-extension path for SCM (Application Composer is used for selected SCM/ERP objects); **REST API** usage (calling Fusion's web services to read/write data programmatically); **Application Composer** (adding custom fields/objects); and **Groovy scripting** (a Java-based scripting language used inside Application Composer for validations, calculated fields, and business logic). Concepts include sandboxes (isolated test environments), configuration vs. personalization vs. extension, and descriptive flexfields (DFFs).
*(c) Why it matters:* Real clients always need changes the standard product doesn't provide. The skill is making them *upgrade-safe*. REST APIs are also the backbone of integration and AI-agent work, so this module bridges into the next clusters.

### Cluster 5 — Integration

**Module 9: XML (27 hrs).**
*(a) What it is:* XML (Extensible Markup Language) is a universal text format for structured data exchange between systems. It's taught before the integration modules because integrations constantly read, transform, and produce XML.
*(b) Topics covered:* Well-formed vs. valid XML documents; **XML Namespaces** (avoiding element-name collisions); **DTD (Document Type Definition)** and **XML Schema (XSD)** (two ways to define and validate document structure, with XSD being the modern, data-typed successor to DTD); **XPath** (a path notation to address/navigate parts of an XML document); **XSLT (Extensible Stylesheet Language Transformations)** (transforming XML from one structure to another); **Oracle JDeveloper** as an XML development tool; **XQuery** (a query language for XML, like SQL for documents); and **Oracle XML DB** (storing/querying XML in the database).
*(c) Why it matters:* BI Publisher reports consume XML; OIC mappers transform JSON/XML; Fusion web services exchange XML. XPath and XSLT skills are used directly inside OIC data mappers.

**Module 10: Integration Basics (18 hrs).**
*(a) What it is:* An architectural introduction to **Oracle Integration Cloud (OIC)**, Oracle's iPaaS (integration platform-as-a-service) for connecting applications.
*(b) Topics covered:* OIC components, features, and capabilities; and **connections and adapters** — prebuilt connectors (REST, SOAP, FTP/SFTP, database, Oracle ERP/HCM, Salesforce, SAP, etc.) that encapsulate authentication and APIs so you don't hand-code connectivity.
*(c) Why it matters:* It's the conceptual primer for the deep OIC module that follows.

**Module 11: Oracle Application Integration Cloud / OIC (54 hrs).**
*(a) What it is:* The deep, hands-on integration module — the program's single largest block, aligned to the Application Integration Professional certification.
*(b) Topics covered:* OIC components; **connections** (the Adapter plus Trigger = the source that starts a flow, and Invoke = a downstream system the flow calls); **Integrations** built in three patterns — **Application** (event/app-driven orchestration), **Schedule** (batch/scheduled runs), and **Event** (publish/subscribe); **Lookups** (key-value mapping tables for code conversions) and the **Data Mapper** (visual field-to-field transformation); **file handling** (bulk file processing); **orchestration actions** (branching, loops, logic); and **fault handling** (error management for resilient flows). The connectivity agent is used to reach on-premises systems. Typical use cases: syncing orders between CRM and Fusion ERP, loading data via FBDI, triggering ESS jobs, extracting BIP report data.
*(c) Why it matters:* Integration is one of the hottest, highest-paid Fusion skills. Almost every client runs Fusion alongside other systems, and OIC is how they talk. This is likely your most marketable technical skill from the program.

### Cluster 6 — The AI Layer

**Module 8: GenWizard (27 hrs).**
*(a) What it is:* **Accenture GenWizard** is Accenture's proprietary, full-suite generative-AI platform for the **Technology Delivery Life Cycle (TDLC)** — i.e., applying gen AI to the work of building and running software itself. Per Accenture's GenWizard page: "Built on Accenture's proven intelligent automation platforms (myWizard®, myNav®, myConcerto), GenWizard conforms to industry standards and Responsible AI principles."
*(b) Topics covered:* Generative AI APIs for practical applications; Gen AI essentials and foundational elements for TDLC use cases — reverse engineering legacy code, generating documentation/a "living knowledge base," code refactoring/re-platforming, data migration, and application/infrastructure management. It integrates traditional AI, automation, and DevOps and adheres to Responsible AI principles.
*(c) Why it matters:* This is Accenture's internal delivery accelerator — you'll likely use it on real projects to speed up Oracle implementations. It explains *how Accenture wants you to work*, infusing AI into every delivery step.

**Module 12: Oracle AI Agent Studio for Fusion Applications Developers (23 hrs).**
*(a) What it is:* Oracle's native, no-cost platform (within Fusion) for building, deploying, and managing **AI agents** — software that can reason, make decisions, and take actions across Fusion Applications, not just answer questions.
*(b) Topics covered:* Defining AI agents; AI Agent Studio's purpose, tools, and security (agents inherit Fusion's existing security/RBAC automatically); designing agents using agent templates, topics, prompts, and tools; building/deploying/testing single-agent and multi-agent teams (supervisor-worker and sequential patterns, with human checkpoints/approvals); embedding agents in Redwood pages or triggering them via events/APIs; and evaluating/troubleshooting agents. Per Oracle's October 15, 2025 announcement (Oracle AI World, Las Vegas), "Oracle AI Agent Studio enables customers and partners to select the best-performing LLM for their business needs, with support for OpenAI, Anthropic, Cohere, Google, Meta, and xAI"; it also supports open standards (MCP, A2A, REST) and RAG for grounding, and Oracle reports more than 32,000 certified experts trained in the tool.
*(c) Why it matters:* This is Oracle's strategic direction for Fusion — "agentic applications." It builds directly on the REST/extensibility skills from Module 7 and the SCM process knowledge from Modules 3–4 (agents automate processes like the Marketplace's "Quote-to-Purchase Requisition Assistant," return-order processing, or requisitioning).

**Module 13: Accenture AI Refinery.**
*(a) What it is:* **Accenture AI Refinery**, unveiled October 2, 2024, is "an AI foundation platform, created by Accenture and built on NVIDIA AI Foundry, NVIDIA AI Enterprise, and NVIDIA Omniverse" (Accenture) for building and deploying **multi-agent AI solutions** across enterprise functions. It is led by Accenture chief AI officer Lan Guan, and the platform has filed 55 patent applications across 10 countries (Accenture, 2025). Where Oracle's AI Agent Studio is Fusion-specific, AI Refinery is cross-platform and vendor-neutral.
*(b) Topics covered:* Building and deploying multi-agent solutions; integrating generative AI and LLMs across enterprise functions; its four pillars — Agents, Knowledge, Models, and Governance. Per Accenture, Models lets you "flex between models using a switchboard that selects the right model based on cost, performance, accuracy or other factors," and the "Distiller Framework... establishes standards for LLM-based, multi-agent solutions" featuring "chain-of-thought reasoning, query transformation for agentic RAG, dynamic task decomposition." It also covers safety features (PII masking, Responsible AI rules) and integration with third-party agents (Amazon Bedrock, Azure AI, Databricks).
*(c) Why it matters:* This positions you to deliver AI across *any* client stack, not just Oracle — Accenture's broader AI play. Together with GenWizard (delivery AI) and AI Agent Studio (Fusion AI), it completes a three-tiered AI skill set.

### The Capstone
A 5-day capstone project plus hands-on assessment requires you to combine the clusters — likely configuring SCM, building a report and an integration, extending a page, and adding an AI agent — simulating a real client engagement.

## Recommendations
- **Stage 1 (Foundations, Modules 1–2):** Lean on your CSE/AI background — ADB and OCI are the most CS-familiar content. Focus on the vocabulary (compartments, IAM, ADW vs. ATP) that recurs everywhere. Benchmark: be able to explain regions/ADs/compartments and provision an ADB unaided.
- **Stage 2 (Functional, Modules 3–5):** This will feel least familiar. Invest here — build a mental model of order-to-cash and procure-to-pay, and the FSM/RBAC structure. Benchmark: explain what happens from order capture to invoice, and the four role types. Without this, the technical modules lack context.
- **Stage 3 (Technical build skills, Modules 6, 7, 9–11):** This is your core marketable skillset. Prioritize OIC (54 hrs, highest weight and market demand) and reporting. Treat XML as the prerequisite grammar for OIC. Benchmark: independently build an OTBI report, a BIP report, and a scheduled OIC integration with a data mapping and fault handler.
- **Stage 4 (AI, Modules 8, 12, 13):** Your AI/ML background is a genuine edge here. Connect each tool to its layer: GenWizard = AI for *delivery*, AI Agent Studio = AI *inside Fusion*, AI Refinery = AI *across the enterprise*. Benchmark: build and test a multi-agent solution in AI Agent Studio.
- **What would change this plan:** If your project assignment is known early (e.g., a pure integration role vs. an analytics role), weight your deepest study toward that cluster. If you struggle with the functional modules, prioritize them over polishing the AI modules, because functional context gates everything else.

## Caveats
- The hour figures and module list come from your Accenture schedule; Oracle's public MyLearn pages confirm the *content* of each learning path but Oracle periodically renames and re-versions paths (e.g., BI Publisher is now "Oracle Analytics Publisher"; ADB is now branded "Autonomous AI Database").
- "GenWizard" and "AI Refinery" are Accenture-proprietary; public detail is largely from Accenture marketing and press releases, so treat capability claims as vendor-reported rather than independently verified — e.g., Accenture reports its PHC Group GenWizard deployment achieved "80% faster ticket resolution, 50% reduced support workload and 30% higher customer satisfaction," and its marketing-function agents yielded a "25-35% reduction in manual steps, 6% cost savings... 25-55% increase in speed to market," all Accenture-reported figures.
- AI Agent Studio and AI Refinery are evolving rapidly (frequent 2025–2026 feature releases), so specific features may shift by the time you train.
- The capstone description is inferred from the program structure; confirm its exact scope with your Accenture trainers.
